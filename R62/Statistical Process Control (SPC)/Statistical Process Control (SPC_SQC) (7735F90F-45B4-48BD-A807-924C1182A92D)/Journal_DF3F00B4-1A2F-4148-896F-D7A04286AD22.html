Monitoring with an SPC Control Chart & Run Rules

Control Chart with Run Rules

SPC Dashboard

Solution:

Access raw data, bringing in the key quality metric and grade code signal (Do it! start search for 'Line 01 Grade Code' in the data tab) 

Identify each grade run, contextualized data with grade runs. (Do it! start use the Condition With Properties tool to create a condition for every time the 'Line 01 Grade Code' changes, name the property 'Grade Code'.) 

  • Note that to remove transition time from each capsule use the Formula Tool and the function shrink() to remove the time in between each grade

Create a control chart using an add-on tool to calculate control limits or manually input them for each grade. Control chart.

  • To choose what data to use to establish control limits there are many options: 
    • Create an in-control condition that identifies the in-control periods to model, be sure to include one from each condition and make sure the time range in the add-on includes all of these capsules. Create an In-Control Condition
    • Establish the time range in the add-on tool used to establish control limits (note that this must include every grade. 
    • Use your own limits: run the add-on tool with a time range, but then edit the control limits formula with a scalar value.

Identify and notify when run rules are violated. (Do it! create an automated email notification for the 'Nelson Run Rule Deviations' condition.)

  • Separate run rule conditions. Note that installing the add-on tool also installs these run rules as individual functions, search 'Run Rules' in the Formula Tool. 
  •  To combine all the run rules into one condition, use the Formula Tool and combinewith() function. Combined Run Rules
  • Nelson Run Rules:

Scale to other lines using asset groups and a Treemap to monitor deviations.

  • Create an asset group after creating the analysis on the first line. First add all items for the first line, save, then add another asset and the unique process variable, condition, and in-control condition, if using one. 
  • Add all items from the asset group onto the trend, and check that the asset group swaps as expected. All items from asset group
  • To create the Treemap, remove the process condition leaving just the combined run rule deviations condition. Then, navigate to Treemap and set the run rule deviation priority color to red. Treemap
