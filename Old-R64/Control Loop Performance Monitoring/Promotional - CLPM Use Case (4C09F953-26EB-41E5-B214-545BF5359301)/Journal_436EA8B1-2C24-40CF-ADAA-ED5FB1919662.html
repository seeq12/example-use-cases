While there are many metrics that may be calculated in CLPM, this Use Case highlights 3 of the most common:

  1. Average Absolute Error (AAE): this metric indicates how well a controller is tracking setpoint and is calculated as the absolute value of the controller error. Workstep
  2. Controller Output Travel: this metric indicates the effort exerted by the controller.  Excessive effort causes additional wear and tear on mechanical elements. Workstep 
  3. % Time in Correct Mode: this metric indicates if the controller is operating in the correct mode (Auto or Cascade), as it was designed. Workstep

 

Each of these metrics must have limits to identify if performance is acceptable.  This limits are determined based upon 1 week of operating data.

Value Search is used to identify when each metric exceeds its limits.

Using an asset structure, these calculations are easily applied to other controllers:

  • FIC008
  • FIC020
  • PIC001
  • PIC002
  • TIC001
  • TIC002
  • TIC012
