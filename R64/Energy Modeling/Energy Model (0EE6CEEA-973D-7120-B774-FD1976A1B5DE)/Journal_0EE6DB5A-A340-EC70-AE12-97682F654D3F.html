Energy Modeling

Challenge: Reduce energy usage at a system level (reactor, line, building) using an overall energy consumption metric (steam header) and all the outputs that impact the overall metric (downstream valves, flow rates, ambient temperature, etc.)

15% Energy Reduction

Solution

Access raw data, bringing in total steam and all tags impacting it. (Do it! start search for 'Line1' in the data tab to find some of the tags used in this analysis)

Cleanse the data, converting string tags to numeric, so they can be used in a regression model. 

  • Use the Formula Tool and the splice() function to splice in 0 or 1 when the valves are open or closed. 

Identify uptime, total steam greater than zero. (Do it! start Use Value Search to find when 'Total Steam' is greater than zero, ignore capsules shorter than 1 day)

Create an energy model of total predicted steam. 

  • Use the Prediction Tool to model Total Steam, input all the rest of the signals as 'Input Signals'
  • Use the Uptime condition to limit the training window to only when there is steam flow
  • Note that regression requires a certain number of variable changes based on the model inputs. If you receive the error, The number of target samples is not sufficient to conduct a regression,'  it is likely due having many flat signals (likely the 0 or 1 signals) that do not change frequently enough to complete the regression matrix (linear algebra related). Some ways to fix 1) change the training window 2) remove 0 or 1 signals that are not required.

Validate the model using a scatterplot over the training window. Trendline: y = 1.11x-1216 (r^2 = 0.900)

  • Navigate to the Scatterplot view and color by the Training Window condition. To create a Training Window Condition, use the Manual Condition Tool and select the time range used to create the Prediction model. 
  • To create a Trendline, use the f(x) button in the toolbar and select 'New Prediction'. Model the Total Steam vs. the Energy Model - Total Steam.

Use the regression coefficients to inform continuous improvement efforts. Then, measure the impact. Trend of Energy Improvements OR Table of Monthly Improvements 

  • Identify where the coefficients indicate that equipment is not working as designed or expected. Flow rates should have a coefficient of 1 (as the flow rate of steam from a valve increases by 1, so should the total steam flow rate). Identify where on/off valves are not operating at design flow rate. 
  • To quantify monthly improvements, calculate predicted steam usage and actual steam usage and find the difference between the two. Then, create a Monthly condition and sue scorecard metric to find the average difference for the month. Trend of monthly improvements
  • Summarize in a monthly improvements condition table.
