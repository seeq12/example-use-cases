Batch Cycle Time - Downtime Identification

Batch Cycle Time Dashboard

Solution:

Access raw data, bringing in key reactor tags. (Do it! start Navigate to the asset group 'Batch Reactors' and add Catalyst Feed Rate and Reactor Temperature.) 

Identify when each batch is active using reactor fill level to identify batches. (Do it! start use Value Search to create a condition for when Reactor Fill Level is greater than 5, ignore capsules shorter than 5 minutes)

Identify the pump in and pump out phases by finding when the derivative is increasing and decreasing.

Identify heat up and catalyst addition phases (Do it! start Use composite condition where A = Increasing Temp after Pump In and B = Catalyst Addition 2 and use A minus B, heat up identified, in lane)

  • Identify the Catalyst Addition phase using the catalyst flow rate. 
  • Take the derivative of temperature to find when temperature is increasing, remove time included in the Pump In phase.  Identify Temperature Increasing 
  • Combine Catalyst Addition and Increasing Temperature after Pump in using Composite Condition and 'A minus B'. Identify Heat Up 
  • All Batch Phases

Identify downtime and quantify batch cycle time (Do it! Create a notification for when downtime occurs.)

  • Identify downtime by subtracting all batch phases from the overall batch condition. 
  • Identify Turnaround time using Formula Tool and the inverse() function.
  • Use Signal From Condition to quantify Cycle Time 

Analyze cycle times and phases with histogram of batch phases.

  • Combine all conditions into one and set capsule properties for each phase using Formula Tool. 
  • Create a histogram measuring the duration of each phase

Quantify cost impact of downtime and scale over all three reactors. Table of downtime events for each reactor

  • Create an asset group to scale calculations.
  • Quantify the total weekly downtime cost, create a weekly condition and use signal from condition to create a signal for the duration of each downtime event and sum over a week. Multiply that by the cost of downtime.
  • Use a scorecard metric to display in a table and the asset button to display over every reactor in the asset group.
