Predict Conveyor Belt Tensioning Requirements Across Conveyor Belts

Problem Statement:

Conveyor belts are a key asset for mining companies. They are utilized to transport solids collected from the mines through various pre-processing steps and eventually transfer the materials to a shipping vessel for transfer to the final customer. Creating a predictive model for conveyor belt tension enables condition-based maintenance scheduling, reducing unnecessary spend on time-based maintenance and catching unplanned belt failures preventing line stoppage events.

Solution:

Monitor Conveyor Belt Network

 

Identify periods of rapidly decreasing tension

  • Identify the tension signals for the belts (assets) in the conveyor belt network, load into the display pane, and visually examine trends in signals over time. Note that different belts will have different beginning tensions, different mean time to failure, varying degrees of signal noise, and different failure dates.
  • Note that even though we have 7 conveyor belts to analyze in our system, we can use Seeq's Asset Tree functionalities to do our analysis on a single asset, then apply it across the entire network of belts. Select one conveyor belt to build the analytics out on. We will use CVB-505.
  • We need to identify the different tension cycles, as well as the end of each cycle when the belt begins to lose tension rapidly. We Identify the change in cycles by calculating the derivative, and searching for when the derivative is high. 
  • We define the cycles as the time between these changes in cycle. 
  • Finally, we identify the time period when the tension is rapidly decreasing as when the derivative is negative.

Project tension into future based on decline period (step to now + 1/2 display forward)

  • Create a model of the tension signal at the end of each cycle. Seeq Formula can be used to create a best fit model based on the data within each "Tension Rapidly Decreasing" capsule. 
  • First, we make a time counter signal that accumulates time during each cycle. 
  • We create the regression condition which stores the intercept and coefficients of the regression equation as capsule properties. 
  • Then we extract these capsule properties as signals and combine those properties with our time counter signal to create the polynomial model equation.
  • Evaluate goodness of fit of the model using Scatter Plot view. 

Detect when tensioning is required based on statistical limits

  • Create the threshold limit that will be used to determine when a belt requires tensioning. In order to scale this across assets which may have different starting tensions and different threshold values, we will define the threshold value as when the tension falls below 50% of the average tension from the first 5 days of the cycle. 
  • We identify the first 5 days, calculate the average tension during that time period, then create a new signal that is equal to 50% of that starting tension.
  • Use value search to identify when the predicted tension will fall below the 50% threshold requiring a tensioning.

Scale across all conveyor belts
(Do it! Swap conveyor from CVB-501 to CVB-502)

  • Scale this analysis to the other conveyor belts in the network. Use Treemap view to identify which belts have been in the Rapidly Decreasing Tension state over the last day.
