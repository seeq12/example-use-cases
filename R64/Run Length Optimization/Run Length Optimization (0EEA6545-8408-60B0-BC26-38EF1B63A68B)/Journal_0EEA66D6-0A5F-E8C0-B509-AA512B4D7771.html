Run Length Optimization

 

Problem Statement

Many manufacturing units hit process throughput constraints over the course of a run. These constraints are often reversible, but at the cost of shutting down to clean/maintain equipment.  

Assuming every production time period has associated downtime, predicting the right time to shutdown and clean equipment enables a unit to meet their production goals sooner. Meeting targets sooner translates into more production (& $$) over time. 

Optimizing run length to meet production target sooner

Solution:

Predict future production rate and identify when the production target of 50,000 klb is met if no downtime is scheduled during the run.

  • Create a capsule for this run - extend the capsule into the future far enough that it will capture the end of the run even with degrading production rates.
  • Calculate the total time elapsed since the beginning of the production run.
  • Create a predictive model of the production rate based on the time signal.
  • Create a combined signal of the Production Rate data that exists and the prediction for the future training.
  • Integrate the projected signal to create a production totalizer. 
  • Identify when 50,000 klb is produced. 

Calculate the number of downtimes that would be required to fulfill the order quantity at any point in the production cycle. 

  • Create a capsule spanning the time over which production rate data exists for this run. 
  • Calculate a totalizer signal for a given amount of product produced at any point during the run. 
  • Calculate the number of downtimes that would be required to fulfill the order quantity at any point in the production cycle. 

Calculate the Average Production Rate at each point during the current run time and the total run time (including downtimes) required to produce the given order quantity based on the number of required downtimes.

  • Calculate the total time elapsed since the beginning of the production run. 
  • Calculate the Average Production Rate at each point during the current run time.
  • Calculate the total run time (including downtimes) required to produce the given order quantity based on the number of required downtimes. 

Find the minimum run length required to show the full order fulfillment time. 

Find the production rate at the point of minimum run length to trigger proactive downtime.

Build a Reference Profile for the ideal production cycles to monitor current operations against.

Quantify the potential time savings and monitor results.
