For CLPM it is important to use an asset structure so that calculations / trending can easily be scaled to many controllers.  Using an asset tree allows users to take advantage of Seeq's Asset Swapping capability and the Treemap visualization.

For CLPM, the following controller tags are of interest:

  • Process Variable (PV): Measured value that is being controlled
  • Setpoint (SP):  this is the target or desired value of the PV
  • Controller Output (CO):  The calculated output value of the controller.  Typically this value is in units of "%" and is transmitted to a Final Control Element (FCE), which is adjusted to maintain the PV at the SP.  Examples of an FCEs include valves, pumps, heaters, etc.
  • Mode:  The mode that the controller is operating in.  Typically, controllers are designed to operated in Auto or Cascade, and it is important to monitor the time that the controller is operating outside of the design mode.
